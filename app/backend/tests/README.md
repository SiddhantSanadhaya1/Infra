## Running Tests

### Install Test Dependencies

```bash
pip install -r requirements-test.txt
```

### Run All Tests

```bash
pytest
```

### Run Specific Test Files

```bash
# Test policy service
pytest tests/services/test_policy_service.py

# Test Lambda worker
pytest tests/test_lambda_worker.py

# Test authentication middleware
pytest tests/middleware/test_auth.py
```

### Run Tests with Coverage

```bash
pytest --cov=src --cov-report=html --cov-report=term
```

### Run Tests in Parallel

```bash
pytest -n auto
```

## Test Patterns

### Boundary Value Analysis
Tests include boundary values for all numeric inputs:
- Minimum values (e.g., age 18, coverage $50,000)
- Maximum values (e.g., age 85, coverage $10,000,000)
- Just below/above boundaries
- Zero and negative values where applicable

### Equivalence Class Partitioning
Tests cover representative values from each input class:
- Valid ranges
- Invalid ranges (too low, too high)
- Edge cases (empty strings, None values)

### Error Condition Testing
Tests verify proper handling of:
- Invalid input data
- Missing required fields
- Business rule violations
- External service failures

### Mocking Strategy
External dependencies are mocked:
- AWS services (S3, SQS, SNS, DynamoDB)
- Database sessions
- Redis cache
- JWT token validation
