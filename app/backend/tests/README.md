## Running Tests

```bash
# Install test dependencies
pip install -r requirements-test.txt

# Run all tests
pytest

# Run with coverage
pytest --cov=src --cov-report=html

# Run specific test file
pytest tests/services/test_policy_service.py

# Run specific test class
pytest tests/services/test_policy_service.py::TestGeneratePolicyNumber

# Run specific test
pytest tests/services/test_policy_service.py::TestGeneratePolicyNumber::test_generate_policy_number_auto
```

## Key Testing Patterns

### Boundary Value Analysis
Tests include boundary conditions for:
- Age limits (18, 85 years)
- Date ranges (policy start/end)
- Claim amounts (coverage limits)
- Fraud scores (risk thresholds)

### Equivalence Partitioning
Tests cover representative values from each input class:
- Valid and invalid dates
- Different policy types
- Various claim statuses
- Multiple fraud risk levels

### Error Handling
Tests verify proper handling of:
- Missing credentials
- Invalid tokens
- Redis/AWS connection failures
- Database errors
- Missing required fields
