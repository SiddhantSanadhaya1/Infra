## Prerequisites

### Running API Instance

**CRITICAL:** These tests require a **running InsureCo API instance**. They do NOT start the service automatically.

#### Option 1: Local Development
```bash
cd app/backend
python -m venv venv
source venv/bin/activate  # On Windows: venv\Scripts\activate
pip install -r requirements.txt

# Set up database (PostgreSQL required)
export DATABASE_URL="postgresql+asyncpg://postgres:postgres@localhost:5432/insureco"

# Start the API
uvicorn src.main:app --reload
```

The API will run at `http://localhost:8000`.

#### Option 2: Docker
```bash

```

#### Option 3: Deployed Environment
Set `BASE_URL` to your deployed API:


### GitHub Actions Example


## Troubleshooting

### Connection Refused

**Problem:** `httpx.ConnectError: [Errno 61] Connection refused`

**Solution:** Ensure the API is running:
```bash
# Check if API is running
curl http://localhost:8000/health

# If not, start it:
cd app/backend
uvicorn src.main:app --reload
```

### Database Errors

**Problem:** `sqlalchemy.exc.OperationalError: could not connect to server`

**Solution:** Ensure PostgreSQL is running and DATABASE_URL is correct:
```bash
# Start PostgreSQL (macOS)
brew services start postgresql@15

# Start PostgreSQL (Linux)
sudo systemctl start postgresql

# Verify connection
psql -U postgres -d insureco_test -c "SELECT 1;"
```

### Import Errors

**Problem:** `ModuleNotFoundError: No module named 'httpx'`

**Solution:** Install test dependencies:
```bash
pip install -r tests/requirements.txt
```

### Test Failures Due to Data Conflicts

**Problem:** `409 Conflict: Email already registered`

**Solution:** Use a fresh test database:
```bash
psql -U postgres -c "DROP DATABASE insureco_test;"
psql -U postgres -c "CREATE DATABASE insureco_test;"
alembic upgrade head
```

## Best Practices

1. **Always use a separate test database** - Never run tests against production data
2. **Run tests before commits** - Catch regressions early
3. **Review failing tests carefully** - Failures may indicate real bugs
4. **Keep tests independent** - Each test should work in isolation
5. **Use meaningful test names** - Names should describe what is being tested
